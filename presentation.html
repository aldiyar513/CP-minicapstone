EventMatch

A lightweight platform for students to host, discover, and join peer-led activities — with reliability tracking, role-based sign-ups, and live ETA built in.

Flask PostgreSQL Leaflet Docker
The Problem

Students want to do things together. Existing tools make it hard.

Finding a study partner, gym buddy, or weekend plan is still routed through group chats and DMs — noisy, unstructured, and full of no-shows.

01

Discovery is broken

Interesting events get buried in group chats. You learn about things after they happened.

02

No accountability

"I'm in" in a chat means nothing. Hosts plan for 10, get 3. Effort wasted.

03

No structure

Who's bringing the ball? Who's driving? Coordination happens in 40 scattered replies.

Gap Analysis

The tools students already use weren't built for this.

Meetup / Facebook Events

  • Designed for strangers and formal events
  • Heavy UI, slow to create spontaneous plans
  • No reliability signal before people join
  • No role assignment or logistics tools

Group chats (WhatsApp / iMessage)

  • Events disappear after a day of scrolling
  • "Maybe" replies never resolve
  • No capacity limit, no waitlist, no ETA
  • Non-members can't discover

EventMatch fills the gap between "formal event platform" and "lossy group chat" — structured enough to trust, light enough to post a 6 PM run at 5:45.

Goals

What we're optimizing for.

Goal 1

Frictionless hosting

A student should be able to post an activity in under 60 seconds, including location pin, roles, and a reliability threshold.

Goal 2

Trust without friction

Reliability scores let hosts filter out no-shows automatically — no reputation system to manually manage.

Goal 3

Structured coordination

Roles, capacity, waitlists, and per-event chat replace the chaos of group DMs.

Goal 4

Free & accessible

Zero API costs: OpenStreetMap instead of Google Maps, free-tier hosting, no billing required to self-host.

What makes it different

Unique features — the ones other tools don't have.

Reliability score gate

Hosts set a minimum reliability threshold (50% / 65% / 80%). Users below it can't join — protects time-sensitive activities.

Role-based sign-ups

Not just "attend." Define roles per event — "driver," "gear," "note-taker" — so logistics are solved before the day of.

Live ETA with privacy

Three tracking modes: visible after joining, host-only, or off entirely. Participants pick their comfort level.

Capacity + waitlist

Events fill, waitlist forms automatically. Someone drops — the next person promotes with zero host work.

Per-event chat

Discussion lives with the event, not in a chat buried between 12 other threads. Disappears when the event ends.

Zero-cost mapping

Leaflet + OpenStreetMap means no API keys, no billing, no quota anxiety. Runs on any free tier.

How it works

From "I want to do something" to "we're on our way."

Step 1

Host posts

Pick category, drop a map pin, set capacity, choose a reliability threshold, define roles.

Step 2

Feed discovers

Students browse by category and date. Reliability filter auto-applied.

Step 3

Join or waitlist

One click. Users pick a role. Capacity full? Auto-waitlist, auto-promote.

Step 4

Coordinate & go

Per-event chat, live ETA on the map, host dashboard tracks who's in and who's late.

Every touchpoint is designed for sub-60-second interactions. No bloat, no ceremony.

Under the hood

Built to be cheap to run, easy to maintain.

Backend

Flask 3 + SQLAlchemy for the ORM. Gunicorn in production with configurable workers/threads. Pytest for regression coverage.

Database

SQLite for local dev, PostgreSQL (via Neon free tier) in production. Automatic URI rewriting handles both.

Maps

Leaflet + OpenStreetMap tiles — no API key, no billing, no quota. Venue pinning and ETA calculated client-side.

Deployment

Docker + Render free tier, kept warm with UptimeRobot pings. Neon for the database. Total monthly cost: $0.
